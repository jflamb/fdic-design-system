# Security Policy

Security issues should not be reported through public GitHub issues.

## Reporting

Use GitHub Private Vulnerability Reporting for this repository when it is enabled.

If private reporting is not yet available, do not post the details publicly. Use an existing private maintainer contact path instead and include `fdic-design-system security report` in the subject or opening line.

## What To Include

Please include:

- a short summary of the issue
- the affected package, file, workflow, or page
- reproduction steps or a proof of concept
- impact assessment
- any suggested mitigation or fix

## Scope

Relevant reports include:

- vulnerable or unsafe dependency changes
- supply chain or workflow integrity issues
- docs-site behavior that could mislead users in a security-sensitive context
- component behavior that could cause trust, privacy, or security failures in high-stakes workflows

## Response

This is a solo-maintainer project, so response times are best effort. The goal is to acknowledge valid reports quickly, coordinate a fix, and disclose publicly only after a mitigation is available or the risk is otherwise understood.
